# Принцип работы шифра Roller Cipher
Шифр Roller Cipher относится к классу потоковых шифров на основе операции XOR с детерминированно генерируемым ключевым потоком (keystream).

## Параметры шифра (ключи)
Шифрование определяется двумя целочисленными параметрами:

`n` — величина циклического сдвига первой последовательности

`m` — величина циклического сдвига второй последовательности

Эти параметры `n` и `m` являются секретными ключами шифра и остаются неизменными для всех операций шифрования в рамках одного сеанса.

## Генерация ключевого потока (`keystream`)
Пусть длина открытого текста составляет `L = len(plaintext)`.

1. Формируются две идентичные базовые последовательности:
`seq = [1, 2, 3, ..., L]`
2. Выполняется циклический сдвиг вправо (right cyclic shift):
    * Первая последовательность сдвигается на n позиций:

        `seq_n = seq[L - n : ] + seq[ : L - n]`

    * Вторая последовательность сдвигается на m позиций:

        `seq_m = seq[L - m : ] + seq[ : L - m]`

        (Примечание: если n или m превышают L, эффективный сдвиг равен n mod L и m mod L соответственно.)

    * Ключевой поток формируется поэлементным сложением:

        `keystream[i] = seq_n[i] + seq_m[i] для всех i = 0, 1, ..., L-1` 
        
        Таким образом, значения keystream лежат в диапазоне от 3 до 2L (включительно).

## Процесс шифрования
Шифрование выполняется побайтово с использованием операции побитового исключающего ИЛИ (XOR):

`ciphertext[i] = ord(plaintext[i]) ⊕ keystream[i]` для `i = 0, 1, ..., L-1`

Результат шифрования представляется в виде списка целых чисел (каждое значение — результат XOR одного символа с соответствующим элементом ключевого потока).

## Процесс расшифрования
Расшифрование симметрично шифрованию благодаря свойству инволюции операции XOR:

`plaintext[i] = ciphertext[i] ⊕ keystream[i]`

Ключевой поток генерируется точно так же, используя те же значения ключей n и m и ту же длину входных данных.

## Пример вычисления (L = 5, n = 2, m = 1)
Базовая последовательность:
`seq = [1, 2, 3, 4, 5]`

Сдвиг вправо на 2 позиции:
`seq_n = [4, 5, 1, 2, 3]`

Сдвиг вправо на 1 позицию:
`seq_m = [5, 1, 2, 3, 4]`

Ключевой поток:
`keystream = [4+5, 5+1, 1+2, 2+3, 3+4] = [9, 6, 3, 5, 7]`

Для открытого текста "hello" (ASCII-коды: 104, 101, 108, 108, 111):

`ciphertext = [104⊕9, 101⊕6, 108⊕3, 108⊕5, 111⊕7] = [97, 99, 111, 105, 104]`

## Основные свойства шифра
* Детерминированность: при фиксированных n, m и длине сообщения keystream всегда одинаков.
* Обратимость: шифрование и расшифрование используют одну и ту же процедуру генерации keystream.
* Зависимость от длины: keystream генерируется заново для каждого конкретного сообщения в зависимости от его длины.
